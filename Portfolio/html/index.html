Life was never the same... Once I started getting into coding.

The endless Youtube tutorials and information out there just sucked me in like a vortex, that I don't think I can get out of and a fact is don't want to. My favorite joke is, that I am without make up like HTML without CSS that's what my life became...
All jokes aside though my motivation for studying was always a chance to improve myself and learn. And coding escpecially is something that is always challenging and exciting.     My role:


  1. Making parts of XD prototype, sketching

    Projects main goals:

  1. To improve and make a mobile version of fronter for students.
  2. Solve existing solution's problems such as:
    • Important information being pushed down
    • Too many steps to get from one location on the page to other
    • Not clear design
    • Website is not responsive

    To complete this project:

    • This project was done in a design sprint week
    • We listened to the client, identified the target audience and issues of excisting solution
    • Created team canvas in order to understand what are the roles, strategies and goals
    • Decided to use google docs for file sharing, messenger for communication and scrum to be clear about what tasks have to be done
    • Brainstormed and mapped out the problem, by defining the target audiences, long term goal and drawing a flowchart
    • By mapping the problem and after asking the experts, we did the HMW notes updated the long term goal and consequently we chose an area to focus on
    • Individually we sketched solutions using crazy 8's sketch method, afterwards sketched solutions were described and voted on
    • Once the sketched solutions were chosen, we drew a storyboard to plan the prototype
    • Created high fidelity prototype parts using Adobe XD adn the sticher stiched the prototype
    • Did the user testing by doing a trunk test and asking user to accomplish some tasks with the created prototype, as well as asked insights on what they are finding while using the prototype
    • After the user testing, we made on cocnlusions on what should be improved and what was done right
