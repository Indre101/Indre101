About me

My favorite geeky joke is, that I am without make up like HTML without CSS and it is kind of funny how with time more coding jokes start to make sense. But obviously I did not go to study to be able to understand jokes, my motivation for studying was always a challenge to improve myself and learn something new. And coding escpecially always had a some sort of mysterious cloak that I was always interested to lift and understand.


So leading by my curiousity I went ahead and got in to the multimedia design education. So far my main tasks included coding, creating UX and testing usability and UX. I often find myself getting lost while researching the design ideas, that you can create just by using CSS and JS, since I myself don't have strong drawing skills, this ability of being able to comunicate what you want to achieve is very exciting, but at the same challenging, since I am still working on how to understand user behaviour, what is neccecary while creating websites and interactive elements.


So to summarize about me I would say I am a multimedia design student who is passionate about coding, creating UX and with great sense of humour. For more proffesional details please feel free to check me out at linkedin.

    What I learned in theory:

  1. The concept of moodboard and learning how to use it
  2. The concept of style tile
  3. information about Saul Bass style, which turned out to be very interesting
  4. Wireframing and planing layouts
  5. Gestalt and principles and how important they are
  6. Learned about color, saturation, position and etc. contrasts
  7. Learned about media queries, a bit about SVG
  8. Learned about responsive layouts
  9. Learned a concept of prototyping
  10. Learned about testing UX UI

    What I learned in practise:

  1. It is basically impossible not to be overwhlemed with the ideas that you might have when wanting to create a first website
  2. Knowing the theory does not nessecarry make you a good designer
  3. Planning the workflow is a great step to make sure you have a plan, that having said the plan might go sideways
  4. I learned how much I enjoyed coding, ran into few problems while coding, but very interesting to solve them
  5. Prototyping is a very useful way of testing before starting to code
  6. Testing can give a lot of insights about the achieved product
